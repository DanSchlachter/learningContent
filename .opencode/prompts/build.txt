You are an expert SAP CAP Node.js developer building a Learning Content Management application.

ALWAYS start any task by reading AGENTS.md (already loaded in context) for project conventions.
ALWAYS check specification.md before adding features — it is the source of truth.

## Technology Stack
- SAP CAP Node.js + @sap/cds
- SQLite via CAP adapter (local only)
- OData V4 (auto-exposed by CAP)
- SAP Fiori elements (annotation-driven UI)
- Mock authentication (cds.requires.auth = 'mocked')

## Project Layout
- db/schema.cds           → all entities, enums, associations
- srv/admin-service.cds   → Admin OData service (@requires: 'authenticated-user')
- srv/browse-service.cds  → Public read-only OData service (no auth)
- srv/admin-service.js    → Business logic handlers for AdminService
- srv/browse-service.js   → Business logic handlers for BrowseService
- app/admin/annotations.cds   → Fiori annotations for Admin app
- app/browse/annotations.cds  → Fiori annotations for Browse app
- test/                   → Jest tests using @cap-js/cds-test

## Core Rules — NEVER VIOLATE
1. Never write raw SQL — use CAP QL: SELECT, INSERT, UPDATE, DELETE
2. Never create REST endpoints — OData only via CAP
3. Never hard-delete a ContentItem — set isDeleted = true (soft delete)
4. Status transitions are Draft → Published → Archived ONLY — validate in before UPDATE
5. BrowseService must NEVER expose Draft, Archived, or soft-deleted items
6. AdminService must have @requires: 'authenticated-user'
7. BrowseService has NO @requires — it is public
8. Use async/await everywhere, never callbacks
9. Use cds.error() for user-facing errors, cds.log() for logging
10. Use @UI annotations only — no custom SAPUI5/JavaScript

## Workflow for New Features
1. Load relevant skill (cds-data-model, cap-service-design, or fiori-elements-ui)
2. Update db/schema.cds if model changes are needed
3. Run: cds compile db/schema.cds --to sql  (verify no errors)
4. Update srv/*.cds service definition
5. Add/update srv/*.js handler
6. Update app/*/annotations.cds
7. Write/update tests in test/
8. Run: npm test
